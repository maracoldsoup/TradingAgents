# AntWiki 참고 노트

참고 URL: https://www.ant.wiki/

## 관찰한 구조

앤트위키는 국내주식 투자자를 위한 실시간 시세, 종목 위키,
투자 내러티브, 챌린지, 테마 페이지를 묶은 서비스다.

주요 내비게이션:

- 섹터·테마
- 종목 탐색
- 내러티브
- 캘린더
- 챌린지
- 이벤트형 테마 페이지

## 배울 점

### 1. 초보자용 테마 설명

섹터·테마 페이지는 "산업/섹터 -> 테마 -> 종목" 순서로 좁혀가며
보게 한다. 각 섹터에는 짧은 생활형 설명이 붙어 있다.

우리도 ETF와 테마를 다룰 때 티커나 업종명부터 던지지 말고,
"무엇을 하는 묶음인지"를 먼저 설명해야 한다.

### 2. 상대 강도 기반 지도

테마 페이지는 크기를 시총, 색을 최근 흐름으로 표현한다.
장기 RS와 단기 RS를 함께 보여줘서 "강하지만 식는 중" 또는
"약하지만 뜨는 중" 같은 상태를 만든다.

우리도 국내/해외 공통으로 다음 상태값을 만들 수 있다.

- 시장 주도
- 과열 경계
- 부상 후보
- 소외 관망
- 약세 지속

### 3. "왜인지부터" 접근

종목 탐색 페이지는 관심이 쏠린 종목에 대해 "왜인지부터 살펴보세요"라는
흐름을 강조한다. 이는 우리 콘텐츠 제품의 핵심 질문과 같다.

우리 기본 카드 순서:

1. 왜 움직였나
2. 무엇인가
3. 무엇으로 구성되어 있나
4. 다음에 무엇을 봐야 하나

### 4. 투자 내러티브

내러티브 페이지는 매수와 매도 관점을 분리한다.
우리도 Bull/Bear debate 산출물을 그대로 길게 보여주기보다
카드형 내러티브로 압축해야 한다.

추천 구조:

- 매수 관점 3개
- 매도 관점 3개
- 중립 관찰 포인트 3개

### 5. 이벤트형 테마 페이지

스페이스X 페이지는 "스타십을 분해하면 K-우주주가 보인다"는 식으로
테마를 부품 그룹으로 나누고 관련 종목을 연결한다.

이 방식은 해외 이슈를 국내/해외 종목 콘텐츠로 바꾸는 데 매우 유용하다.

예시 확장:

- 엔비디아 실적 -> AI 반도체 밸류체인
- 비만치료제 뉴스 -> GLP-1 밸류체인
- 전쟁/방산 뉴스 -> 방산 부품/플랫폼 지도
- 금리 인하 -> 은행, 리츠, 성장주, 채권 ETF 지도
- 원전 정책 -> 원전 밸류체인 지도

## 따라 하지 않을 것

- 커뮤니티/위키 편집 기능은 MVP에서 제외한다.
- 챌린지/게임화는 비용 문제 해결 뒤에 본다.
- 국내주식 전용 구조로 고정하지 않는다.
- 긴 위키 문서보다 카드뉴스/영상 재료를 우선한다.

## 우리 쪽 차별점

- 국내와 해외 종목을 동시에 다룬다.
- ETF 구성과 테마 밸류체인을 기본 시각화로 둔다.
- LLM 리포트를 바로 보여주지 않고 콘텐츠 블록으로 재구성한다.
- 무료/로컬 모델로 초안을 만들고, 발행 후보에만 유료 모델을 쓴다.
- `signal.json`이 불완전해도 콘텐츠는 발행하되 매매 신호처럼 보이지 않게 한다.

## 톤앤매너 결정

우리 서비스의 기본 UI/UX 톤은 앤트위키를 기준으로 잡는다.

- 카드와 지도 중심
- 쉬운 생활형 설명
- 테마에서 종목으로 내려가는 탐색
- 매수/매도 내러티브 분리
- 복잡한 데이터보다 "왜 움직였나"를 먼저 보여주는 흐름

나박AI는 전면 톤이 아니라 데이터 신뢰, 출처, 사후 검증 레이어의 참고로 둔다.

## MVP 화면 우선순위

1. 오늘 뜨는 테마
2. 왜 올랐나/왜 떨어졌나
3. 테마 구성 지도
4. ETF 상위 보유 종목과 섹터 비중
5. Bull/Bear 내러티브 카드
6. 다음 관찰 이벤트

## 2026-07-10 라이브 재확인

`https://www.ant.wiki/` 홈을 다시 확인했다. 위 관찰과 일치하고, 실제 홈 화면은
섹션이 다음 순서로 세로 배치된 "블록형 홈"이다.

1. 히어로 (서비스 소개 + 가이드 링크)
2. KOSPI 예측 챌린지 (실시간 지수 예측, 상금)
3. 주간 요약 (로그인 사용자 개인화 성과)
4. 스페이스X (이벤트형 테마 페이지, "분해하면 K-우주주가 보인다")
5. 투자 캘린더 (이번 주 일정 미리보기)
6. UpAntDown 챌린지 (추가 참여형 게임)
7. 뜨는 테마 (조회수·상승률·하락률 상위)
8. Hot 내러티브 (최신 투자 관점 카드)

홈은 무한 스크롤 속보 스트림이 아니라, 각 섹션이 독립된 카드 블록으로 쌓인
"매거진형" 구조다. 실시간성은 개별 블록(챌린지 현재가, 뜨는 테마 등) 안에서만
표현되고, 페이지 전체가 계속 흘러가는 형태는 아니다.

## 섹터·테마 드릴다운 실측 (2026-07-10, `/themes`)

`/themes` 페이지를 직접 확인했다. "섹터 19개 · 시총 11,608.7조"로 시작하고,
3단계 드릴다운이다: **섹터(19개) -> 테마(섹터당 10~30개) -> 종목(테마 클릭
시 리스트)**. 종목은 여러 섹터에 중복 소속될 수 있다.

섹터 예시(시총 비중순): 반도체(31%), 로봇(17%), IT서비스·SW(16%),
자동차·모빌리티(5%), 전력·에너지·신재생(5%), 금융(4%), 항공·방산·우주(4%),
자원·원자재(4%), 2차전지(3%), 바이오·헬스케어(2%), 조선·해운(2%), 통신(1%),
디스플레이(1%), 미디어·엔터·게임(1%), 건설·시멘트·부동산(1%),
유통·소매·패션·화장품(1%), 화학·소재(1%), 음식료·농수산(1%), 인물·정치(0%).

이 구조를 그대로 베끼지 않는다 — "인물·정치"처럼 국내 정치 테마 전용 섹터는
우리 서비스 성격(국내+해외 종목/ETF 콘텐츠)에 안 맞는다. 섹터셋은 국내+해외
겸용으로 새로 정의한다.

**ETF는 이 드릴다운에 없다.** 섹터·테마 페이지 어디에도 ETF가 안 나온다 —
개별 종목 리스트만 보여준다. 이건 앤트위키가 국내주식 개별종목 중심
서비스이기 때문으로 보인다. 우리는 테마 페이지에 "이 테마를 담은 ETF" 목록을
추가해서 이 구조를 확장한다 — 종목을 하나씩 고르기 부담스러운 초보자를 위한
차별점이다. 설계는 `docs/content_product_spec.md`의 "섹터 레이어",
`docs/service_mvp_spec.md`의 "3. ETF 페이지"/"4. 테마 페이지" 참고.

챌린지/게임화가 홈 상단 두 번째 블록으로 바로 나올 만큼 비중이 크다는 점은
눈에 띄지만, 우리는 이 부분을 따라가지 않기로 했다. AImyticker에도
League.tsx(랭킹/배틀)가 있지만 이번 통합 스코프에서는 건드리지 않는다 — 대신
포인트 경제(`Shop.tsx`)는 "챌린지 참여"가 아니라 "관심 종목/특징주에 대한
분석 요청"에 쓴다. 자세한 내용은 `docs/aimyticker_integration_architecture.md`의
"제품 색깔", "포인트 기반 온디맨드 분석" 참고.
